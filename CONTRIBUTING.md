# 🤝 Contribuyendo a App Blueprint Generator — Contributing Guide

¡Gracias por tu interés en participar! Este ecosistema ha sido diseñado bajo una visión **Human-Centric**. Creemos fervientemente en el Desarrollo Aumentado: queremos que la Inteligencia Artificial actúe como un amplificador de la capacidad técnica humana, no como un reemplazo ciego. 

---

## 🌟 Cómo puedes ayudar / How to help

### 1. Danos una Star ⭐
Es la forma más directa de apoyar el crecimiento del framework y ayudarnos a llegar a más profesionales independientes y equipos de ingeniería.

### 2. Reportar Errores / Report Bugs
Si encuentras un fallo en el entorno gráfico, rutinas asíncronas o el inyector Python:
1. Abre un **[Issue](https://github.com/jesusgarciafernandez/app-blueprint-generator/issues)**.
2. Describe el error y tu OS.
3. Adjunta logs o capturas de pantalla si bloquea el generador.

### 3. Sugerir Mejoras / Suggest Improvements
¿Falta soporte de orquestación para nuevos LLMs? ¿Crees que la inyección de Sistema Multi-Agente (MAS) es mejorable? ¡Abre un ticket con la propuesta!

### 4. Aportar "Skills" al Repositorio / Contribute Skills
Si tienes maestría táctica en alguna faceta de programación, diseño, marketing o bases de datos, propón ese módulo dentro de las carpetas abstractas `/REPOSITORIO/`. 
Recuerda el estándar fundamental:
- Toda Skill debe contener un archivo `SKILL.md` (Puro Manual Técnico de Reglas).
- Debe acompañarse de su `workflow.md` (La estandarización o paso a paso de uso).
- Mantén la neutralidad tecnológica transversal en su definición.

---

## 🛠️ Estándares Técnicos / Technical Standards
- **Core de la Aplicación:** HTML5, Vanilla CSS, JS (Zero Dependencies), Python 3.6+.
- **Idioma:** Preferiblemente bilingüe (ES/EN) o profesional en español para el core principal y la bóveda de conocimientos.
- **Licencias:** Contribuyendo, acatas la licencia dual del repositorio: MIT expansiva para el software y motor, y CC BY-NC-SA 4.0 para proteger colectivamente la táctica de conocimiento cognitivo de las Skills.
