## Descripción
Esta habilidad permite descomponer problemas complejos en subtareas ejecutables en paralelo por una flota de agentes especializados. Al crear "enjambres" (swarms) de agentes con misiones específicas y contextos aislados, se maximiza la velocidad de desarrollo y se reduce el riesgo de errores por saturación de contexto en un solo agente.

## Cuándo usarla
- **Proyectos de gran escala**: Cuando una sola sesión de IA se vuelve lenta o pierde coherencia.
- **Auditorías o refactorizaciones masivas**: Para auditar seguridad, testear UI y refactorizar código simultáneamente.
- **Tareas repetitivas pero complejas**: Para procesar múltiples archivos o componentes de forma independiente.

## Requisitos
- Sistema de gestión de agentes con capacidad de spawneo.
- Directorios de trabajo aislados (ej: `./.swarm/`).
- Archivos de misión (`MISSION.md`) claramente definidos para cada sub-agente.
