## Descripción
Habilidad técnica fundamental y laboriosa en el pipeline de datos que se ocupa de transformar, mapear y limpiar datos crudos desde su formato original desordenado hacia un formato útil para el análisis y el modelado. El Data Wrangling (o Data Munging) consume hasta el 80% del tiempo de un proyecto de Ciencia de Datos y es el determinante directo de la precisión de cualquier informe o algoritmo posterior ("Garbage In, Garbage Out"). Esta skill incluye el manejo de valores nulos, la unificación de formatos de fecha, la corrección de errores tipográficos masivos, la reconstrucción de datos faltantes y la normalización de esquemas provenientes de múltiples fuentes.

## Cuándo usarla
Escenarios que activan esta skill:
- Antes de cargar datos de múltiples archivos Excel manuales en un Data Warehouse centralizado.
- Al recibir exportaciones de bases de datos antiguas con codificaciones de caracteres erróneas o campos inconsistentes.
- Para unificar bases de datos de clientes (CRM) donde el mismo nombre aparece escrito de 5 formas distintas.
- Durante la integración de datos de APIs externas que devuelven estructuras JSON complejas y anidadas.
- Como paso previo obligado a cualquier entrenamiento de Machine Learning o visualización en BI.

## Requisitos
- Maestría en Python (Pandas/NumPy) o Lenguaje R (dplyr/tidyr).
- Dominio de Expresiones Regulares (Regex) para limpieza de texto complejo.
- Conocimiento de técnicas de imputación estadística (media, moda, regresión).
- Habilidad en SQL para transformaciones masivas en base de datos.
- Pensamiento lógico para detectar patrones de error y "sanity checks".
