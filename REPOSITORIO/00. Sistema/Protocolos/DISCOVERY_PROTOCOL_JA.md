# 🛸 Blueprint Discovery Protocol ディスカバリー・プロトコル：あなたのビジョン、私たちの技術

👋 **始める前に（非常に重要）**

このドキュメントは**技術的なものではありません**。プログラミングの知識も、人工知能の理解も、複雑なデジタルツールの使用経験も必要ありません。

👉 必要なのはたった一つ：**「あなたの現状」を知ることです。**

まだアプリケーションを構築する段階ではありません。それよりもずっと重要なこと、つまり**「本当に何が必要か」をじっくり考えること**から始めます。これが明確でないと、テクノロジーは役に立たず、自動化は失敗し、時間とお金が無駄になってしまうからです。

🎯 **このドキュメントを書き終えると、以下のものが得られます：**
*   本当の問題が完全に明確になります。
*   解決策の構造化されたアイデアが得られます。
*   **Antigravity** エンジンがあなたのアプリケーションを正しく構築するための強固な基盤が整います。

🧠 **回答のしかた：**
*   **普段話すように答えてください：** 完璧な答えを探したり、技術用語を使おうとしたりしないでください。
*   **明確かつ正直に：** このドキュメントはあなたを評価するためではなく、あなたを理解するために設計されています。

**🎯 目的：** 「どうやって」作るかを考えずに、最終的な結果をイメージすること。

> [!NOTE]
> **技術的な補足：** アプリに入力するもの（メールなど）を **インプット (Input)**、出てくる結果（PDFや通知など）を **アウトプット (Output)** と呼びます。

1.  **完璧なデジタルアシスタントがいたら、何をしてほしいですか？**
    *   *以下の文を完成させてください：* 「[特定の出来事] が起きたとき、アプリが自動的に [この作業] をして、その後に私に [この結果] を届けてほしい。」

2.  **あなた自身がコントロールし続けたいことは何ですか？**
    *   *機械が勝手に決めてはいけないことを知ることは非常に重要です。*
    *   **回答欄：** _________________________________________________

3.  **解決策をどのように「見たい」ですか？**
    *   *注意：* これを**インターフェース (UI/UX)** と呼びます。

## 📚 プロジェクトのための小さな辞書

| 私たちが使用する技術用語 | 日常生活での意味 |
| :--- | :--- |
| **Blueprint** (ブループリント) | アプリケーションの「全体図」や DNA のようなもの。 |
| **Skill** (スキル) | 私たちの AI がすでに持っている「能力」（例：WhatsApp を送信するなど）。 |
| **Input / Output** (入出力) | アプリに入る「材料」と、そこで「製造」されて出てくるもの。 |
| **Dashboard** (ダッシュボード) | すべてを一目で確認できる、メインの操作パネル。 |
