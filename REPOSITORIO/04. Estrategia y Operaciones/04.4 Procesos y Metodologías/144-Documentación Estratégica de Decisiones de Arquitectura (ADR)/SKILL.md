## Descripción
Esta habilidad preserva la inteligencia y el Porqué tras cada decisión técnica. Un ADR (Architecture Decision Record) es un documento breve que captura una decisión arquitectónica junto con su contexto y las consecuencias de haberla tomado. El profesional utiliza esta práctica para evitar la amnesia técnica, facilitar el onboarding de nuevos miembros y asegurar que la evolución del sistema sea coherente y fundamentada.

## Cuándo usarla
- **Elección de Tecnologías Críticas**: Para justificar por qué se eligió un lenguaje, base de datos o framework sobre otros.
- **Definición de Patrones de Diseño**: Para registrar el consenso sobre cómo estructurar el código o las comunicaciones entre servicios.
- **Refactorización de Grandes Sistemas**: Para documentar el camino recorrido y evitar repetir errores pasados.

## Requisitos
- Repositorio de código (Git) o wiki centralizada.
- Plantilla estándar de ADR (ej: Michael Nygard template).
- Herramientas de control de versiones.
