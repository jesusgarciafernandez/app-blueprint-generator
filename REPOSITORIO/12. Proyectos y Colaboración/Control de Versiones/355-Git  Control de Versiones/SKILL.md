## Descripción
Habilidad esencial para la administración, seguimiento y colaboración en el desarrollo de software mediante el sistema de control de versiones distribuido Git. Esta skill abarca desde los fundamentos de los commits y ramas hasta técnicas avanzadas de resolución de conflictos, rebase interactivo y estrategias de flujo de trabajo (GitFlow, GitHub Flow). Enseña a mantener un historial de cambios limpio, gestionar repositorios remotos, realizar revisiones de código (Code Reviews) mediante Pull Requests y garantizar la integridad del código fuente en entornos de equipo. El objetivo es proporcionar un control total sobre la evolución del proyecto y facilitar la integración continua.

## Cuándo usarla
Escenarios que activan esta skill:
- Siempre que se inicie un nuevo proyecto de software, independientemente de su tamaño.
- Al trabajar en equipos multidisciplinares donde varias personas modifican el mismo código simultáneamente.
- Para realizar copias de seguridad incrementales y versionadas de cualquier tipo de archivo de texto (ej: configuraciones, documentación).
- Cuando es necesario revertir el sistema a un estado anterior debido a un error crítico detectado en producción.
- Durante la integración de tareas de desarrollo con sistemas de automatización (CI/CD).

## Requisitos
- Git instalado en el sistema local.
- Cuenta en una plataforma de hosting (ej: GitHub, GitLab, Bitbucket).
- Cliente de terminal (bash/zsh) o cliente gráfico (ej: GitKraken, Sourcetree).
- Conocimiento básico de estructura de archivos y directorios.
- Claves SSH configuradas para acceso seguro a repositorios remotos.
