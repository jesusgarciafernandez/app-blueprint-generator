## Descripción
Habilidad de evaluación de interfaces basada en la inspección experta utilizando principios de diseño de interacción establecidos, conocidos como "heurísticas". El Análisis Heurístico permite identificar problemas de usabilidad en una interfaz sin necesidad de realizar pruebas directas con usuarios en etapas tempranas. Se fundamenta principalmente en las 10 Reglas Heurísticas de Jakob Nielsen (visibilidad del estado del sistema, relación entre el sistema y el mundo real, control y libertad del usuario, etc.). Esta disciplina proporciona un diagnóstico rápido, económico y altamente efectivo para mejorar la fluidez de navegación y reducir la curva de aprendizaje de cualquier producto digital.

## Cuándo usarla
Escenarios que activan esta skill:
- En las primeras fases de un rediseño para detectar fallos estructurales de usabilidad antes de iterar visualmente.
- Como paso previo a un test con usuarios para eliminar errores "obvios" y centrar el test en problemas más profundos.
- Al realizar auditorías competitivas para evaluar la facilidad de uso de los productos de la competencia.
- Tras el desarrollo de una nueva funcionalidad para asegurar que cumple con los estándares de interacción de la plataforma.
- Cuando el presupuesto o el tiempo no permiten realizar una investigación con usuarios completa pero se necesita validar la calidad de la interfaz.

## Requisitos
- Conocimiento profundo de las 10 Heurísticas de Usabilidad de Nielsen.
- Familiaridad con los principios de diseño de interacción y psicología cognitiva aplicada a interfaces.
- Capacidad analítica para separar la estética personal de la funcionalidad sistémica.
- Habilidad para documentar hallazgos de forma estructurada, asignando niveles de severidad a cada problema.
- Conocimiento de patrones de diseño estándar (UI Patterns) para web y móvil.
