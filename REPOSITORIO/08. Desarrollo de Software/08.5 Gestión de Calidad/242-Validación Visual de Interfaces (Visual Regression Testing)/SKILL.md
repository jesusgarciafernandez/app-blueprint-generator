## Descripción
Esta habilidad es el "ojo biónico" de la garantía de calidad. El profesional de Visual Regression Testing automatiza el proceso de verificar que cada cambio en el código no rompa inadvertidamente la apariencia de la interfaz. Utiliza herramientas que capturan imágenes de la aplicación en diversos dispositivos y las comparan píxel a píxel con una "línea base" (Baseline) aprobada. Se enfoca en detectar desplazamientos, cambios de color o tipografía que las pruebas funcionales tradicionales no pueden ver. Es el guardián de la perfección estética.

## Cuándo usarla
- **Actualización de Sistemas de Diseño (Design Systems)**: Para asegurar que un cambio en un componente base no afecte negativamente a cientos de pantallas.
- **Desarrollos Multi-dispositivo y Multi-navegador**: Para garantizar la coherencia visual absoluta en todas las resoluciones y entornos.
- **Proyectos de Alta Exigencia Estética (Luxury/Premium)**: Donde la marca no se puede permitir ni el más mínimo error visual.

## Requisitos
- Herramientas de regresión visual (Applitools, Percy, Playwright Snapshots, BackstopJS).
- Conocimientos de CSS, Layouts de UI y jerarquía visual.
- Integración en pipelines de CI/CD para una validación continua.
