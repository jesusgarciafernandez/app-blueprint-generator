## Descripción
Esta habilidad es el lenguaje universal de la arquitectura moderna. El modelo C4 (Contexto, Contenedores, Componentes, Código) proporciona una forma estructurada y poderosa de visualizar sistemas complejos sin perderse en el detalle excesivo ni quedarse en la abstracción inútil. El profesional utiliza estos cuatro niveles de zoom para asegurar que cada stakeholder (desde el CEO hasta el Desarrollador Senior) entienda la parte del sistema que le corresponde.

## Cuándo usarla
- **Diseño de Nuevos Sistemas**: Para definir las fronteras de responsabilidad clara desde el inicio.
- **Documentación de Sistemas Heredados (Legacy)**: Para extraer el conocimiento y facilitar el mantenimiento.
- **Revisiones de Arquitectura y Seguridad**: Para identificar puntos de fallo o vulnerabilidades a través de una visión clara de las interconexiones.

## Requisitos
- Herramientas de diagramación (Draw.io, Lucidchart, PlantUML, Mermaid.js).
- Conocimientos profundos de la jerarquía C4 propuesta por Simon Brown.
- Capacidad de abstracción técnica y comunicación efectiva.
