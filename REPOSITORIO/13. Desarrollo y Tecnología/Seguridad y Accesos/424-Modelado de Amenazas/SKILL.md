## Descripción
Habilidad estratégica y técnica para identificar, clasificar y mitigar posibles riesgos de seguridad en una aplicación o infraestructura antes de que se escriba la primera línea de código o se realice el despliegue. Esta skill se basa en metodologías como STRIDE (Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service, Elevation of Privilege) y el análisis de flujos de datos (DFD). Permite a los equipos de desarrollo y seguridad anticipar vectores de ataque, evaluar el impacto de posibles brechas y priorizar las defensas necesarias siguiendo el principio de Seguridad por Diseño (Security by Design). El objetivo es construir sistemas resilientes que minimicen la probabilidad de explotación y reduzcan el coste de corregir fallos de seguridad en etapas tardías.

## Cuándo usarla
Escenarios que activan esta skill:
- Al iniciar el diseño de una nueva arquitectura de software o infraestructura cloud para integrar controles de seguridad desde el principio.
- Para evaluar el impacto de seguridad al introducir nuevas funcionalidades críticas (ej: cambios en el sistema de autenticación).
- Durante la fase de auditoría de seguridad para documentar los riesgos conocidos y las medidas de mitigación implementadas.
- Cuando se requiere educar a los equipos de desarrollo sobre posibles vectores de ataque específicos de su aplicación.
- Para cumplir con requisitos regulatorios que exigen una evaluación formal de riesgos y amenazas.

## Requisitos
- Conocimientos de metodologías de modelado (STRIDE, PASTA, VAST).
- Capacidad para crear y leer Diagramas de Flujo de Datos (DFD) y diagramas de arquitectura.
- Entendimiento de vectores de ataque comunes (OWASP Top 10).
- Familiaridad con herramientas de modelado (Microsoft Threat Modeling Tool, OWASP Threat Dragon).
- Experiencia en el análisis de riesgos y priorización de vulnerabilidades.
