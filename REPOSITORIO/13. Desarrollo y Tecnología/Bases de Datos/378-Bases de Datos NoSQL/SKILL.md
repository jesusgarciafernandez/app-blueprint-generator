## Descripción
Habilidad técnica avanzada centrada en el uso de sistemas de gestión de datos no relacionales diseñados para casos de uso específicos: alta disponibilidad, escalado horizontal masivo y manejo de datos no estructurados. Esta skill abarca diversos modelos NoSQL: Documentales (MongoDB), Clave-Valor (Redis), Columnares (Cassandra) y de Grafos (Neo4j). Enseña a diseñar esquemas flexibles basados en patrones de acceso (Query-Driven Design), gestionar el Teorema CAP (Consistencia, Disponibilidad, Partición) y optimizar el rendimiento mediante sharding y replicación. El objetivo es proporcionar soluciones de persistencia de altísima velocidad para aplicaciones modernas que manejan Big Data.

## Cuándo usarla
Escenarios que activan esta skill:
- Al desarrollar aplicaciones que necesiten manejar volúmenes masivos de datos en tiempo real (ej: redes sociales, IoT).
- Para implementar sistemas de caché de alto rendimiento que reduzcan la latencia de respuesta del servidor.
- Durante el diseño de catálogos de productos con atributos altamente variables.
- Al trabajar con datos jerárquicos o redes complejas de relaciones (Grafos).
- Cuando el escalado vertical de una base de datos relacional se vuelve económicamente inviable.

## Requisitos
- Motores de base de datos instalados o acceso Cloud (ej: MongoDB Atlas, Redis Cloud).
- Bibliotecas de conexión/drivers para el lenguaje de programación elegido.
- Conocimientos sólidos de JSON y modelado de documentos.
- Comprensión de conceptos de red distribuidos y particionamiento de datos.
- Herramientas de visualización (ej: MongoDB Compass, RedisInsight).
