## Descripción
Habilidad técnica fundamental para el diseño, implementación y mantenimiento de sistemas de gestión de bases de datos relacionales (RDBMS). Esta skill abarca desde el modelado conceptual y lógico (Normalización) hasta la escritura de consultas SQL complejas y optimizadas. Se centra en garantizar la integridad referencial, el cumplimiento de las propiedades ACID (Atomicidad, Consistencia, Aislamiento y Durabilidad) y el rendimiento óptimo mediante el uso inteligente de índices, vistas y procedimientos almacenados. El objetivo es construir almacenes de datos robustos, seguros y eficientes que sirvan como fuente de verdad única para aplicaciones empresariales.

## Cuándo usarla
Escenarios que activan esta skill:
- Al diseñar el esquema de datos para una aplicación transaccional compleja (ej: E-commerce, sistema financiero).
- Para optimizar consultas lentas que están degradando el rendimiento de una aplicación en producción.
- Durante la implementación de sistemas de reporting y analítica que requieran JOINs avanzados y funciones de ventana.
- Al configurar políticas de backup, réplicas de lectura y alta disponibilidad de datos.
- Cuando es necesario migrar datos entre sistemas heterogéneos garantizando la integridad de las relaciones.

## Requisitos
- Motores de base de datos instalados (ej: PostgreSQL, MySQL o MariaDB).
- Herramientas de visualización y consulta (ej: DBeaver, pgAdmin, DataGrip).
- Conocimiento profundo del lenguaje SQL (DDL, DML, DCL).
- Comprensión de los niveles de normalización (1NF a 3NF).
- Familiaridad con el análisis de planes de ejecución (EXPLAIN).
