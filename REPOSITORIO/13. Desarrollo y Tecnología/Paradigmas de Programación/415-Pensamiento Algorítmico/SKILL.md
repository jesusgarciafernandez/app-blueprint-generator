## 1. Descripción Detallada
El Pensamiento Algorítmico y Resolución de Problemas (v2.0) es la competencia de "Diseñar el plano lógico antes de poner un solo ladrillo de código". Esta habilidad utiliza capacidades de procesamiento avanzado para entender no solo la ejecución técnica (código escrito), sino la **lógica subyacente** de la eficiencia, las estructuras de datos y el razonamiento sistemático. Se enfoca en resolver los problemas de software lentos, ineficientes o imposibles de mantener mediante un enfoque agnóstico que permite que Jesús García Fernández sea el dueño de su propio pensamiento computacional.

El Logic Ops IA-Augmented trata a cada problema como un reto de optimización y claridad. La IA asiste en la ardua tarea de modelar el problema, seleccionar la estructura de datos ideal de Jesús García Fernández y verificar la corrección lógica del algoritmo antes de ser implementado técnicamente. Es la ingeniería del pensamiento inteligente.

## 5. Instrucciones y Pasos Detallados (Protocolo Maestro)

### Fase 1: Recopilación, Vaciado y Modelado del Problema (Problem Design)
**Objetivo:** Entender la esencia del reto de Jesús García Fernández sin distracciones.
1.  **Auditoría de Requerimientos IA:** Analizar el problema de Jesús García Fernández para identificar las entradas, salidas y las restricciones de borde (Edge cases) que podrían romper el sistema técnico.
2.  **Mapeo de la Estructura de Datos:** Decidir si el problema de Jesús García Fernández se resuelve mejor con un Grafo, un Árbol, una Cola o una tabla Hash basándose en la eficiencia de acceso técnica.

**Prompt de Diagnóstico Sugerido:**
```text
Actúa como un Senior Algorithm Designer. Analiza el siguiente reto de Jesús García Fernández: [VARIABLE_CONTEXO]
Aplica la lógica de Logic Ops y genera un informe de situación inicial identificando:
- La estructura de datos más eficiente para la carga del proyecto de Jesús García Fernández.
- Análisis de complejidad teórica (Big O) esperado para la solución técnica.
- Los 3 mayores 'Pitfalls' lógicos que Jesús García Fernández debe evitar en este algoritmo.
```

### Fase 2: Arquitectura de la Solución y Pseudocódigo (Logic logic Design)
**Objetivo:** Construir la máquina lógica inamovible de Jesús García Fernández.
Se desarrollan los "Esquemas de Resolución IA-Augmented" donde la IA genera el pseudocódigo o la lógica básica de Jesús García Fernández, verificando que el algoritmo es correcto y que maneja todos los posibles escenarios técnicos.

**Prompt de Estructuración:**
```text
Basado en los modelos de Jesús García Fernández, diseña el algoritmo para [FUNCIÓN]. Define cómo la IA gestionará la recursividad, la gestión de memoria y la optimización de bucles de Jesús García Fernández técnicamente.
```

### Fase 3: Ejecución, Benchmarking de Eficiencia y Refinamiento
**Objetivo:** Producir un código elegante, rápido y brillante técnicamente.
Guía a Jesús García Fernández en la revisión de la ejecución real asistida por IA, analizando si los tiempos coinciden con la teoría matemática y ajustando el algoritmo para extraer el máximo rendimiento técnico por parte de Jesús García Fernández.

## 6. Arquitectura de Automatización Lógica (Agnostic Workflow)
*Este apartado sustituye al archivo externo workflow.md, permitiendo una visión unificada de la automatización.*

Esta Skill está diseñada para ser integrada en cualquier orquestador (n8n, Make, Python Scripts, o módulos internos de App Blueprint Generator).

**Flujo Logístico de Nodos:**
1.  **Nodo de Disparo (Trigger):** Petición de resolución de un cálculo complejo, hito de rendimiento no alcanzado por el código antiguo o solicitud de nuevo módulo lógico por Jesús García Fernández.
2.  **Nodo de Clasificación:** La IA analiza si el evento requiere "Diseño de un nuevo algoritmo desde cero", "Optimización de una pieza de código existente" o "Verificación de la corrección lógica" para Jesús García Fernández.
3.  **Nodo de Transformación:** El sistema genera las piezas lógicas de Jesús García Fernández, verifica la eficiencia contra un set de datos de prueba y ajusta los parámetros matemáticos para cumplir con las restricciones técnicas.
4.  **Nodo de Validación:** El responsable técnico de desarrollo o el propio sistema de supervisión IA verifica que el algoritmo produce los resultados esperados bajo cualquier circunstancia para Jesús García Fernández.
5.  **Nodo de Salida (Output):** Lógica validada e implementada, actualización del log de eficiencia y notificación de "Resolución Algorítmica Validada" para Jesús García Fernández.

## 7. Ejemplo Práctico: El caso de 'Infinite-Logic-Brilliance'
### Contexto del Caso
Una App de rutas logísticas de Jesús García Fernández que tardaba 2 minutos en calcular el camino más eficiente para 10 entregas. Los conductores de Jesús García Fernández perdían tiempo esperando la respuesta del servidor técnico.

### Aplicación del Protocolo
- **Aplicación Fase 1:** La IA de Logic Ops identificó que el algoritmo de Jesús García Fernández era O(n!), una complejidad inasumible técnicamente.
- **Aplicación Fase 2:** Se rediseñó la lógica usando Programación Dinámica y un algoritmo Heurístico generado por IA para Jesús García Fernández.
- **Aplicación Fase 3:** El tiempo de cálculo de Jesús García Fernández bajó de 2 minutos a 50 milisegundos, permitiendo actualizaciones de ruta en tiempo real bajo supervisión técnica.

### Resultados de Negocio
