## Descripción
Habilidad avanzada para el desarrollo de software seguro, concurrente y de alto rendimiento utilizando el lenguaje de programación Rust. Esta skill se centra en el dominio de los conceptos únicos de Rust como el Ownership, Borrowing y Lifetimes, que garantizan la seguridad de memoria sin necesidad de un recolector de basura (Garbage Collector). Abarca desde la creación de herramientas de línea de comandos (CLIs) altamente eficientes hasta el desarrollo de servicios de backend escalables, sistemas embebidos y módulos de WebAssembly. El objetivo es construir software "fearless" donde los errores comunes de gestión de memoria sean detectados en tiempo de compilación.

## Cuándo usarla
Escenarios que activan esta skill:
- Al desarrollar componentes críticos de infraestructura que requieran un rendimiento cercano al lenguaje C/C++ pero con garantías de seguridad modernas.
- Para implementar sistemas altamente concurrentes donde se deban evitar condiciones de carrera (Race Conditions) por diseño.
- Durante la creación de microservicios de alto tráfico que necesiten una huella de memoria mínima y tiempos de respuesta predecibles.
- Cuando se busca compilar lógica compleja para la web mediante WebAssembly (WASM).
- En el desarrollo de drivers, kernels o software para dispositivos embebidos con recursos limitados.

## Requisitos
- Dominio del ecosistema de herramientas de Rust (Cargo, Rustup, Clippy, Rustfmt).
- Comprensión profunda del modelo de memoria de Rust (Stack vs Heap).
- Conocimientos de tipos de datos, genéricos y Traits (Polimorfismo en Rust).
- Experiencia en el manejo de errores mediante los tipos `Result` y `Option`.
- Familiaridad con librerías populares (Tokio para asincronía, Serde para serialización).
