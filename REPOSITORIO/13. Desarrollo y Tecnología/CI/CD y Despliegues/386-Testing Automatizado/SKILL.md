## Descripción
Habilidad estratégica centrada en el aseguramiento de la calidad mediante la creación de suites de pruebas programáticas que validan el comportamiento del software de forma autónoma. Esta skill abarca la pirámide de pruebas completa: desde Unit Tests (lógica aislada), Integration Tests (comunicación entre módulos) hasta End-to-End Tests (flujos de usuario reales en el navegador). Utiliza frameworks modernos (Jest, Playwright, Cypress) para detectar regresiones de forma instantánea, garantizar la estabilidad de la lógica de negocio y mejorar la confianza en los despliegues continuos. El objetivo es reducir drásticamente los fallos manuales y asegurar una experiencia de usuario impecable.

## Cuándo usarla
Escenarios que activan esta skill:
- Antes de subir cualquier cambio a ramas de producción para asegurar que no se rompe la funcionalidad existente.
- Durante el desarrollo de APIs críticas que requieran validación estricta de contratos de datos.
- Al implementar flujos de autenticación y pasarelas de pago que no pueden fallar bajo ninguna circunstancia.
- Para realizar pruebas de carga y estrés sobre sistemas que esperan picos repentinos de tráfico.
- Durante procesos de refactorización de código heredado (Legacy) para garantizar la preservación del comportamiento.

## Requisitos
- Frameworks de testing instalados (ej: Jest para Unit, Playwright para E2E).
- Conocimientos de lenguajes de scripting (JavaScript, Python o similar).
- Entorno de desarrollo con datos de prueba controlados (Mocks/Stubs).
- Acceso a un pipeline de CI (ej: GitHub Actions) para ejecución automática.
- Comprensión de selectores CSS/XPath para pruebas en interfaces web.
