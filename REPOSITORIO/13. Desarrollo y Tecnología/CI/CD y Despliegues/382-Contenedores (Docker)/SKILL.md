## Descripción
Habilidad técnica avanzada centrada en la creación, gestión y despliegue de aplicaciones utilizando contenedores Docker. Esta skill enseña a empaquetar código, dependencias y configuraciones en imágenes inmutables que garantizan un comportamiento idéntico en desarrollo, testing y producción. Abarca la optimización de `Dockerfiles` mediante builds multietapa (multi-stage), la orquestación local con `Docker Compose`, el manejo de volúmenes para persistencia de datos y la configuración de redes aisladas. El objetivo es eliminar el problema de "en mi máquina funciona" y acelerar los ciclos de entrega de software mediante la portabilidad y el aislamiento de procesos.

## Cuándo usarla
Escenarios que activan esta skill:
- Cuando el equipo de desarrollo necesita un entorno de trabajo estandarizado e independiente del sistema operativo anfitrión.
- Al implementar arquitecturas de microservicios que requieran aislamiento total entre componentes.
- Para simplificar el pipeline de CI/CD, permitiendo que el mismo artefacto probado sea el que se despliegue.
- Durante la modernización de aplicaciones legadas (Legacy Migration) mediante su "contenedorización" para facilitar la portabilidad cloud.
- Al testear aplicaciones complejas que dependan de múltiples servicios (DBs, Caches, Proxies) de forma efímera.

## Requisitos
- Motor de Docker instalado (Docker Desktop o Docker Engine).
- Conocimientos de CLI (interfaz de línea de comandos).
- Comprensión de conceptos básicos de sistemas operativos Linux (permisos, procesos, variables de entorno).
- Registro de imágenes configurado (Docker Hub, AWS ECR, GitHub Packages).
