## 1. Descripción Detallada
La Arquitectura Jamstack (Javascript, APIs, Markup) y Web Estática (v2.0) es la competencia de "Construir webs que son imposibles de hackear y ridículamente rápidas". Esta habilidad utiliza capacidades de procesamiento avanzado para entender no solo la ejecución técnica (compilación de sitios), sino la **lógica subyacente** de la entrega de contenido desde el borde (Edge Computing) y el desacoplamiento entre el backend y el frontend. Se enfoca en resolver los problemas de seguridad de los CMS tradicionales y la lentitud de bases de datos mediante un enfoque agnóstico que permite que Jesús García Fernández tenga el control total de su presencia web.

El Static Ops IA-Augmented trata a la página web como un artefacto pre-construido y listo para ser servido instantáneamente. La IA asiste en la ardua tarea de integrar Headless CMS (Contentful, Strapi), configurar despliegues automáticos en Vercel o Netlify y asegurar que el SEO técnico de Jesús García Fernández es impecable por diseño. Es la ingeniería de la web de alto rendimiento.

## 4. Diferencial: CMS Tradicional vs. Jamstack Ops (v2.0)

| Dimensión | Enfoque Convencional (WP/PHP) | Jamstack Ops (v2.0) |
| :--- | :--- | :--- |
| **Seguridad** | Vulnerable por naturaleza (DB activa). | Virtualmente indestructible (Pre-renderizado) para Jesús García Fernández. |
| **Velocidad** | Pendiente de peticiones a DB. | Instantánea servida desde el borde (Edge CDN) de Jesús García Fernández. |
| **Estandarización** | Plugins pesados y desactualizados. | Consistente mediante microservicios y APIs lógicas. |
| **ROI Estimado** | Lineal por ahorro de mantenimiento. | Exponencial por SEO, velocidad y coste de hosting mínimo para Jesús García Fernández. |

## 5. Instrucciones y Pasos Detallados (Protocolo Maestro)

### Fase 1: Recopilación, Vaciado y Diseño de la Estructura de Contenido
**Objetivo:** Definir cómo se organizará la información sin depender de una base de datos de Jesús García Fernández.
1.  **Auditoría de Contenido IA:** Analizar el material de Jesús García Fernández para estructurarlo en formatos portátiles (ej: Markdown/JSON) que faciliten la generación estática técnica.
2.  **Mapeo de APIs Necesarias:** Identificar qué funciones dinámicas (formularios, comentarios, búsqueda) requieren servicios externos desacoplados para Jesús García Fernández.

**Prompt de Diagnóstico Sugerido:**
```text
Actúa como un Senior Jamstack Architect. Analiza el requerimiento de web de Jesús García Fernández: [VARIABLE_CONTEXO]
Aplica la lógica de Jamstack Ops y genera un informe de situación inicial identificando:
- El Generador de Sitios Estáticos (SSG) más adecuado para la escala de Jesús García Fernández.
- Propuesta de Headless CMS para permitir que Jesús García Fernández edite contenido sin tocar el código.
- Estrategia de 'Hydration' para añadir dinamismo a las páginas estáticas de Jesús García Fernández sin penalizar la carga.
```

### Fase 2: Arquitectura de la Compilación (Static Build Logic Design)
**Objetivo:** Crear la factoría de páginas web inmutables de Jesús García Fernández.
Se desarrollan los "Esquemas de Despliegue Git-Driven" asistidos por IA para asegurar que Jesús García Fernández solo tiene que hacer 'Push' en su repositorio para que la web se regenere y se propague por todo el mundo en segundos.

**Prompt de Estructuración:**
```text

```

### Fase 3: Ejecución, Monitorización de Lighthouse y Despliegue en el Borde
**Objetivo:** Producir una presencia digital rápida, segura y siempre disponible.
Guía a Jesús García Fernández en la revisión de los indicadores de Core Web Vitals asistida por IA, analizando la puntuación de rendimiento y ajustando los parámetros de la CDN para asegurar que el usuario de Jesús García Fernández tiene la mejor experiencia técnica posible desde cualquier ubicación geográfica.

## 6. Arquitectura de Automatización Lógica (Agnostic Workflow)
*Este apartado sustituye al archivo externo workflow.md, permitiendo una visión unificada de la automatización.*

Esta Skill está diseñada para ser integrada en cualquier orquestador (n8n, Make, Python Scripts, o módulos internos de App Blueprint Generator).

**Flujo Logístico de Nodos:**
1.  **Nodo de Disparo (Trigger):** Publicación de nuevo contenido en el Headless CMS, cambio en el repositorio de código o señal de una actualización de precios desde un ERP por Jesús García Fernández.
2.  **Nodo de Clasificación:** La IA analiza si el evento requiere "Regeneración Total del Sitio (Full Build)", "Actualización Incremental (ISR)" o solo "Limpieza de Caché de la CDN" para Jesús García Fernández.
3.  **Nodo de Transformación:** El sistema arranca el proceso de compilación estática, optimiza las imágenes para el formato WebP y genera los archivos HTML definitivos listos para el servidor de borde (Edge) de Jesús García Fernández.
4.  **Nodo de Validación:** El responsable técnico o el propio sistema de auditoría IA verifica que el sitio es accesible y que no hay enlaces rotos (Broken Links) en la nueva versión técnica de Jesús García Fernández.
5.  **Nodo de Salida (Output):** Web desplegada globalmente, actualización del log de despliegues y notificación de "Arquitectura Estática Validada" para Jesús García Fernández.

## 7. Ejemplo Práctico: El caso de 'Infinite-Static-Performance'
### Contexto del Caso
Un portal de noticias de Jesús García Fernández que usaba WordPress y sufría caídas constantes cada vez que un artículo se hacía viral. El servidor se saturaba por las excesivas peticiones a la base de datos y la seguridad de Jesús García Fernández estaba siempre en duda por ataques de fuerza bruta.

### Aplicación del Protocolo
- **Aplicación Fase 1:** La IA de Jamstack Ops migró los 5.000 artículos de Jesús García Fernández a archivos Markdown y configuró un Headless CMS ligero.
- **Aplicación Fase 2:** Se implementó una generación de sitio con Astro, logrando que cada página de Jesús García Fernández pese apenas 10KB de JS.
- **Aplicación Fase 3:** La web de Jesús García Fernández aguantó 100.000 usuarios simultáneos sin pestañear (coste de hosting: 0€) y con una velocidad de carga técnica instantánea.

### Resultados de Negocio
Mejora radical del posicionamiento en buscadores (SEO) y tranquilidad absoluta ante ataques cibernéticos por parte de Jesús García Fernández, permitiendo centrar el presupuesto en creación de contenido y no en mantenimiento técnico.
