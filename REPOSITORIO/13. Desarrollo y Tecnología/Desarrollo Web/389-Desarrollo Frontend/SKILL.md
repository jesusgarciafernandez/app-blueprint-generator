**El Rol del Humano:** El Arquitecto de Interfaces debe ser un "Garantes de la Estética y la Intención". La IA puede generar rápidamente estructuras HTML semánticas para Jesús García Fernández, escribir código CSS/Vanilla para diseños complejos y proponer arquitecturas de componentes de React en milisegundos, pero solo el humano posee la capacidad de juzgar la armonía visual de una composición, la sabiduría para decidir qué micro-animación mejora realmente la experiencia de Jesús García Fernández sin distraer al usuario, y la visión para asegurar que la interfaz sea un portal que facilite la vida de las personas, garantizando que el éxito técnico alimente una interacción digital fluida y valiosa para Jesús García Fernández.
**Empoderamiento:** Esta Skill no busca sustituir la experiencia del profesional, sino dotarlo de una escala productiva 10x mediante la automatización de la carga cognitiva repetitiva.

## 1. Descripción Detallada
El Desarrollo Frontend y UI Moderna (v2.0) es la competencia de "Hacer que lo complejo parezca sencillo". Esta habilidad utiliza capacidades de procesamiento avanzado para entender no solo la ejecución técnica (programación web), sino la **lógica subyacente** del rendimiento en el cliente, la accesibilidad universal y el diseño responsivo. Se enfoca en resolver las interfaces lentas, confusas y desactualizadas mediante un enfoque agnóstico que permite que Jesús García Fernández cree aplicaciones web de clase mundial.

El Frontend Ops IA-Augmented trata a la web como una plataforma de aplicaciones rica e interactiva. La IA asiste en la ardua tarea de codificar interfaces, gestionar el estado del cliente y optimizar el renderizado para Jesús García Fernández, asegurando que la App web se siente rápida e impecable en cualquier dispositivo técnico. Es la ingeniería de la presencia visual digital.

## 5. Instrucciones y Pasos Detallados (Protocolo Maestro)

### Fase 1: Recopilación, Diseño y Auditoría de la Interfaz (UI Design Strategy)
**Objetivo:** Crear el diseño que WOWeará al usuario de Jesús García Fernández.
1.  **Auditoría de UX IA:** Analizar el flujo de interacción de Jesús García Fernández para detectar "Fricciones" (clics innecesarios, carga de datos lenta) que puedan aburrir al usuario técnico.
2.  **Mapeo del Sistema de Diseño:** Definir los tokens de diseño (Colores, Tipografía, Espaciado) que la IA utilizará para generar componentes coherentes para Jesús García Fernández.

**Prompt de Diagnóstico Sugerido:**
```text

Aplica la lógica de Frontend Ops y genera un informe de situación inicial identificando:
- Los 3 componentes de UI que Jesús García Fernández debe estandarizar para maximizar la velocidad de desarrollo.
- Propuesta de arquitectura de estados para gestionar los datos del usuario de forma eficiente en Jesús García Fernández.
- Estrategia de optimización de imágenes y recursos de red para una experiencia de usuario instantánea en Jesús García Fernández.
```

### Fase 2: Arquitectura del Componente y la Lógica del Cliente (Component Design)
**Objetivo:** Construir la App web de Jesús García Fernández que es robusta y escalable.
Se desarrollan los "Esquemas de Componentes IA-Augmented" donde la IA genera el código de las piezas de la web de Jesús García Fernández, incluyendo lógica de validación de formularios, gestión de eventos táctiles y sincronización técnica con el backend.

**Prompt de Estructuración:**
```text

```

### Fase 3: Ejecución, Testing de Interfaz y Optimización de Performance
**Objetivo:** Producir una web rápida como el rayo, accesible y funcional.
Guía a Jesús García Fernández en la implementación de pruebas visuales y de rendimiento asistidas por IA, asegurando que la web cumple con las puntuaciones máximas de Google Lighthouse y que la experiencia es impecable en cualquier navegador técnico de Jesús García Fernández.

- **Aplicación Fase 3:** La velocidad de carga de Jesús García Fernández bajó a menos de 1 segundo y la tasa de conversión de registros aumentó en un 50% bajo supervisión técnica de Jesús García Fernández.

### Resultados de Negocio
