## Descripción
Habilidad estratégica enfocada en el diseño, implementación y gestión de arquitecturas de sistemas escalables, altamente disponibles y seguros en los principales proveedores de nube (AWS, Microsoft Azure y Google Cloud Platform). Esta skill abarca desde el diseño de redes virtuales (VPCs) hasta la selección de servicios de cómputo, almacenamiento y bases de datos que optimicen el rendimiento y el costo según las necesidades del negocio. Se enfoca especialmente en los principios "Cloud-Native" para garantizar que las aplicaciones aprovechen al máximo la elasticidad y resiliencia de la nube.

## Cuándo usarla
Escenarios que activan esta skill:
- Cuando el usuario requiere diseñar una infraestructura para una aplicación con alta demanda variable que necesite auto-escalado.
- Durante la fase de migración de aplicaciones on-premise a un entorno de nube pública (Rehosting, Replatforming o Refactoring).
- Al planificar una estrategia de recuperación ante desastres (Disaster Recovery) distribuida geográficamente (Multi-AZ o Multi-Region).
- Cuando se necesite realizar una optimización de costos (FinOps) en infraestructuras existentes bajo el Well-Architected Framework.

## Requisitos
- Acceso a una cuenta de proveedor Cloud (AWS, Azure o GCP) con permisos de administrador o IAM específicos.
- Conocimientos profundos de redes (VPC, Subnets, Security Groups, Peering, Transit Gateway).
- Dominio de herramientas de Infraestructura como Código (IaC) como Terraform, Pulumi o CloudFormation.
- Familiaridad con arquitecturas de microservicios y comunicación asíncrona (Event-driven).
