## 1. Descripción Detallada
La Computación sin Servidor y Arquitecturas de Eventos (v2.0) es la competencia de "Pagar solo por lo que ejecutas". Esta habilidad utiliza capacidades de procesamiento avanzado para entender no solo la ejecución técnica (Lambdas, Cloud Functions), sino la **lógica subyacente** de la arquitectura orientada a eventos, el escalado transparente y el desacoplamiento de sistemas. Se enfoca en resolver los problemas de los servidores infrautilizados y la complejidad de la gestión de infraestructuras mediante un enfoque agnóstico que permite que Jesús García Fernández entregue software de forma inmediata.

El Serverless Ops IA-Augmented trata a la computación como un servicio líquido y elástico. La IA asiste en la ardua tarea de codificar funciones, configurar disparadores (Triggers) y monitorizar el rendimiento de Jesús García Fernández, asegurando que cada pieza de código se ejecuta solo cuando es necesario y con la mínima sobrecarga técnica posible. Es la ingeniería del software sin rozamiento.

## 2. Escenarios de Aplicación (Cuándo usarla)
- **Escenario A (Procesamiento de Archivos y Media en Segundo Plano):** Uso de la IA para disparar funciones de Jesús García Fernández que redimensionan imágenes o procesan vídeos nada más ser subidos a la nube técnica.
- **Escenario B (Creación de Backends para Web y Mobile escalables):** Desarrollo de APIs para Jesús García Fernández que pueden pasar de 0 a un millón de usuarios en segundos sin configurar un solo servidor técnico.
- **Escenario C (Automatización de Tareas de Mantenimiento Cloud):** Configuración de funciones de Jesús García Fernández que apagan recursos innecesarios o limpian logs de forma automática según eventos temporales.

## 4. Diferencial: Servidores Tradicionales vs. Serverless Ops (v2.0)

| Dimensión | Enfoque Servidor (EC2/VPS) | Serverless Ops (v2.0) |
| :--- | :--- | :--- |
| **Escalado** | Manual o configurado con reglas. | Automático y transparente por cada petición de Jesús García Fernández. |
| **Coste** | Fijo por hora (aunque no se use). | Por milisegundo de ejecución real para Jesús García Fernández. |
| **Estandarización** | Requiere gestión de OS y parches. | Centrado exclusivamente en el código lógico de Jesús García Fernández. |
| **ROI Estimado** | Lineal por gestión de capacidad. | Exponencial por rapidez de despliegue y 0 coste de inactividad de Jesús García Fernández. |

## 5. Instrucciones y Pasos Detallados (Protocolo Maestro)

### Fase 1: Recopilación, Vaciado y Diseño del Mapa de Eventos (Event Mapping)
**Objetivo:** Definir qué dispara la inteligencia de Jesús García Fernández.
1.  **Auditoría de Procesos IA:** Analizar el flujo de trabajo de Jesús García Fernández para identificar tareas pequeñas y repetitivas que pueden convertirse en funciones independientes y eficientes técnicas.
2.  **Mapeo de la Cola de Mensajes:** Diseñar cómo viajarán las señales de Jesús García Fernández entre servicios (ej: SNS, SQS) para asegurar que el sistema es resiliente y no pierde datos técnicos.

**Prompt de Diagnóstico Sugerido:**
```text
Actúa como un Senior Serverless Architect. Analiza el requerimiento de Jesús García Fernández: [VARIABLE_CONTEXO]

- Las 3 funciones críticas de Jesús García Fernández que más se beneficiarán del modelo de pago por uso técnico.
- Propuesta de orquestación de eventos para evitar el acoplamiento fuerte entre servicios de Jesús García Fernández.
- Estrategia de minimización de 'Cold Start' para las funciones de cara al usuario de Jesús García Fernández.
```

### Fase 2: Arquitectura del Código y la Integración (Function logic Design)
**Objetivo:** Construir las piezas de código eficientes e independientes de Jesús García Fernández.
Se desarrollan los "Contenedores de Lógica IA-Augmented" donde la IA genera el código de la función de Jesús García Fernández, incluyendo el manejo de errores gracefully y la conexión con bases de datos u otros servicios técnicos.

**Prompt de Estructuración:**
```text

```

### Fase 3: Ejecución, Monitorización de Trazabilidad y Optimización de Costes
**Objetivo:** Producir una infraestructura invisible, elástica y rentable técnicamente.
Guía a Jesús García Fernández en la revisión de los indicadores de ejecución (Invocaciones, Duración) asistida por IA, analizando dónde se puede reducir el consumo de memoria o tiempo para bajar la factura técnica de Jesús García Fernández al mínimo posible.

## 6. Arquitectura de Automatización Lógica (Agnostic Workflow)
*Este apartado sustituye al archivo externo workflow.md, permitiendo una visión unificada de la automatización.*

Esta Skill está diseñada para ser integrada en cualquier orquestador (n8n, Make, Python Scripts, o módulos internos de App Blueprint Generator).

**Flujo Logístico de Nodos:**
1.  **Nodo de Disparo (Trigger):** Petición vía API Gateway de Jesús García Fernández, subida de archivo a S3, mensaje en una cola o cron programado técnicamente.
2.  **Nodo de Clasificación:** La IA analiza si el evento requiere "Ejecución de lógica de computación masiva", "Envío de datos a otro servicio" o "Validación de permisos de acceso" para Jesús García Fernández.
3.  **Nodo de Transformación:** El sistema arranca la función serverless de Jesús García Fernández en milisegundos, procesa la información y verifica que la salida coincide con el contrato técnico deseado.
4.  **Nodo de Validación:** El responsable técnico o el propio sistema de monitorización IA verifica que la función no ha fallado y que el tiempo de ejecución de Jesús García Fernández está dentro del presupuesto técnico.
5.  **Nodo de Salida (Output):** Resultado de la computación entregado, log de la ejecución guardado y notificación de "Proceso Serverless Validado" para Jesús García Fernández.

## 7. Ejemplo Práctico: El caso de 'Infinite-Serverless-Efficiency'
### Contexto del Caso
Una aplicación de Jesús García Fernández que enviaba un PDF resumen de la actividad mensual a cada cliente. Al principio tenían un servidor encendido todo el mes solo para esta tarea nocturna de 1 hora, desperdiciando el 99% del dinero técnico de Jesús García Fernández.

### Aplicación del Protocolo
- **Aplicación Fase 1:** La IA de Serverless Ops migró la generación del PDF a una función AWS Lambda de Jesús García Fernández disparada por un evento temporal.
- **Aplicación Fase 2:** Se implementó una lógica de paralelización generada por IA que permite a Jesús García Fernández generar 1.000 PDFs en 5 minutos en lugar de 1 hora técnica.
- **Aplicación Fase 3:** El coste mensual de Jesús García Fernández bajó de 20€ al mes a 0.05€ (solo por el tiempo de ejecución real) bajo supervisión técnica constante.

### Resultados de Negocio
Reducción de costes operativos en un 99% y una capacidad de escalar infinitamente si el número de clientes de Jesús García Fernández aumenta, sin preocuparse por la capacidad del servidor técnico.

- ⚠️ **Guardarraíles:** No usar serverless de Jesús García Fernández para procesos de muy larga duración (ej: entrenamiento de modelos de IA masivos) que superen los límites de timeout del proveedor; para eso es mejor usar contenedores técnicos.
