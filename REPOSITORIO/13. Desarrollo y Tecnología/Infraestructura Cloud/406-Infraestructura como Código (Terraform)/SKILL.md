## Descripción
Habilidad técnica especializada en la definición, despliegue y gestión de infraestructura tecnológica mediante archivos de configuración declarativos utilizando HashiCorp Configuration Language (HCL). Esta skill abarca el ciclo de vida completo de los recursos cloud (cómputo, redes, bases de datos) en múltiples proveedores (AWS, Azure, GCP). Enseña a gestionar estados remotos de forma segura, el uso de módulos para la reutilización de código, la implementación de planes de ejecución para verificar cambios antes de aplicarlos y la integración de políticas de seguridad preventivas. El objetivo es eliminar la gestión manual de infraestructura, garantizando entornos replicables, versionados y consistentes.

## Cuándo usarla
Escenarios que activan esta skill:
- Al configurar infraestructuras complejas que deben ser replicadas en múltiples entornos (Dev, Staging, Prod).
- Durante la migración de sistemas on-premises a la nube mediante automatización de redes y cómputo.
- Para implementar infraestructuras temporales para pipelines de CI/CD que se crean y destruyen automáticamente.
- Cuando se requiere auditar y versionar todos los cambios realizados en la infraestructura del sistema.
- Al gestionar infraestructuras multi-cloud donde se necesite un lenguaje común para provisionar recursos.

## Requisitos
- Herramienta CLI de Terraform instalada y configurada.
- Credenciales de acceso programático al proveedor cloud (AWS IAM, Azure Service Principal, etc.).
- Conocimientos sólidos de redes cloud (VPCs, Subnets, Security Groups).
- Manejo de sistemas de control de versiones (Git) para almacenar el código de infraestructura.
- Comprensión de conceptos de State Management y Backend remotos (S3, GCS, Azure Blob).
