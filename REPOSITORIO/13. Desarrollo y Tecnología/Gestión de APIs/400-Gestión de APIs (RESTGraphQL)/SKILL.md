## Descripción
Habilidad especializada en el diseño, implementación, documentación y mantenimiento de interfaces de programación de aplicaciones (APIs) eficientes y fáciles de consumir. Esta skill abarca las arquitecturas RESTful (basadas en recursos y verbos HTTP) y GraphQL (basadas en consultas flexibles y esquemas fuertemente tipados). Enseña a crear contratos de API claros, gestionar el ciclo de vida de los servicios (versionado), asegurar la interoperabilidad entre sistemas heterogéneos y garantizar la seguridad mediante Rate Limiting y autenticación. El objetivo es facilitar la integración sin fricciones entre el frontend, aplicaciones móviles y servicios externos.

## Cuándo usarla
Escenarios que activan esta skill:
- Al diseñar la capa de comunicación para un sistema de microservicios distribuido.
- Cuando una empresa necesita exponer sus servicios o datos a desarrolladores externos (Public APIs).
- Para optimizar la carga de datos en aplicaciones móviles mediante GraphQL para evitar el over-fetching.
- Durante la transición de sistemas monolíticos a arquitecturas basadas en servicios mediante el diseño "API-First".
- Al implementar pasarelas de pago o integraciones complejas con software empresarial (ERP/CRM).

## Requisitos
- Herramientas de diseño y testeo (ej: Postman, Insomnia, Swagger/OpenAPI).
- Conocimientos de protocolos de red (HTTP, HTTPS, JSON, XML).
- Servidor de aplicaciones o Gateway (ej: Kong, Tyk, AWS API Gateway).
- Dominio de modelado de datos y esquemas GraphQL (SDL).
- Esquemas de autenticación (API Keys, JWT, OAuth2).
